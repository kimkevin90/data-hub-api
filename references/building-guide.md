# 건물정보 가이드

## 1. 언제 이 카테고리를 쓰나

건물정보는 단지, 동, 호, 건물 상세 속성을 확인할 때 사용한다.

| 목적 | 추천 상품 | 기준 키 |
|---|---|---|
| 단지 기본 속성 | 주거형 단지 기본정보 | `complex_key`, `name`, `region_code` |
| 단지 통합 보기 | 주거형 단지 통합 정보 | `complex_key`, `complex_name`, bbox |
| 단지 내부 동/건물 통합 | 주거형 동 정보 통합 | `complex_key`, `ppk`, bbox |
| 동·호·층·면적 상세 | 주거형 동호 상세 | `complex_key` |
| 주소/PNU 기반 건물 상세 | 건물 상세 정보 | `pnu`, `ppk`, `jpk`, `address` |

## 2. 일반 흐름

```text
단지명 -> 단지명 검색 -> pnu/표준 단지명 후보
pnu 또는 단지명 후보 -> 건물 상세/단지 기본 정보 보강
complex_key 확보 후 -> 단지 기본/통합 정보
complex_key -> 동 정보 통합 또는 동호 상세
```

```text
주소/PNU -> 건물 상세 정보
건물 상세 정보의 pnu -> 필지 중심 좌표
건물 상세 정보의 ppk -> 주거형 동 정보 통합 -> complex_key
complex_key 확보 후 -> 단지/실거래 상품
```

## 3. 주의점

- 건물 상세 정보(`tb_building_total`)에는 좌표가 없다. 지도 표시는 `pnu`로 좌표 상품을 추가 조회한다.
- 주거형 동호 상세은 대용량 상품이므로 `complex_key`와 `limit` 없이 호출하지 않는다.
- 주거형 단지 통합 정보는 단지 기본정보, polygon, marker 계열 값을 한 행에 묶은 상품이다.
- `dp_complex_info`의 세대수·좌표·최고층 계열은 기본정보 출처와 polygon 출처 컬럼이 나란히 존재하므로 자동 합산하지 않는다.
- `dong_id`는 동호 상세 원천 동 코드이며 `ppk/jpk/pnu` 직접 연결키가 아니다.
