# Domain Keys

## 1. 키 요약

| 키 | 의미 | 사용 위치 | 주의 |
|---|---|---|---|
| `complex_key` | 주거형 단지 식별 축 | 단지, 단지 polygon, PPK 마커, 실거래 통합 | 아파트 ID 자체가 아니라 상품 간 연결 키 |
| `pnu` | 필지 식별자 | 건물 상세, 필지 좌표, 실거래, 공시가격 | 19자리 문자열. 숫자 변환 금지 |
| `ppk` | 건물 표제부 식별자 | 동 정보, PPK 마커, 공시가격, 시세 | 단독 유일성 가정 금지 |
| `jpk` | 전유부 식별자 | 건물 상세, 공시가격, 시세 | `none`은 NULL이 아니라 문자열 sentinel |
| `legaldong_code` | 법정동 코드 | 법정동 polygon, 지역 기반 실거래 | `region_code`와 지역 연결 축으로 사용 |
| `region_code` | 단지 상품의 지역 코드 | 단지 기본/통합 검색 | prefix 검색 가능 |
| `building_management_number` | 건축물대장 건물관리번호 | 건물 polygon | `pnu`/`ppk`와 다른 키 체계 |
| `dong_id` | 단지 내부 동 코드 | 주거형 동호 상세 | `ppk/jpk/pnu` 직접 연결키가 아님 |

모든 키는 문자열로 다룬다. 앞자리 0 손실, 숫자 정렬, 암묵적 캐스팅을 피한다.

## 2. 일반 흐름

```text
단지명 -> 단지명 검색 -> pnu/표준 단지명 후보
complex_key 확보 후 -> 단지 기본/통합, 단지 polygon, PPK 마커, 실거래

주소 또는 PNU -> 건물 상세 -> pnu/ppk/jpk
ppk -> 주거형 동 정보 통합 -> complex_key 후보
pnu -> 필지 중심 좌표, 실거래 건물 마커, 실거래 이력
ppk -> PPK 중심 좌표, 동 정보 통합, 공시가격/시세
```

## 3. 조인 주의

- 같은 키 이름이라도 테이블별 row 단위가 다를 수 있다.
- `pnu` 1개가 여러 건물, 거래, 사업자 row로 늘어날 수 있다.
- `ppk`는 상품에 따라 `jpk`, 기준월, 단지키와 함께 봐야 한다.
- 생활인프라 레이어는 대부분 직접 연결키가 아니라 bbox 또는 거리 계산으로 결합한다.
- 건물 상세 정보에는 좌표가 없으므로 지도 표시는 `pnu`로 좌표 상품을 추가 조회한다.
